## 📋 調査完了 - 状況の説明

ご指摘の通りです！先ほどのテストスクリプトで起動したエージェントは、MAOのダッシュボードやログビューアには表示されません。

### 🔍 理由

MAOのエージェントがダッシュボードに表示されるには：

1. **`StateManager`への登録が必須**
   - `.mao/agent_states.db`（SQLite）に保存
   - メモリ内の辞書（`_states`）にも保持

2. **`AgentListWidget.update_agent()`の呼び出しが必須**
   - ダッシュボードのUIに反映

先ほどのテストでは：
- ✅ `AgentExecutor`の作成 → 成功
- ✅ ログファイルの作成 → 成功（`test_logs_init/`）
- ❌ **`StateManager`への登録** → していない
- ❌ **ダッシュボードへの通知** → していない

つまり、**独立したプロセスとして起動されたが、MAOのエージェント管理システムには登録されていない**状態でした。

## ✅ 正しいテスト方法

### 方法1: インタラクティブモード経由（推奨）

```bash
mao start "エージェント起動テスト: 5つのロールを試す"
```

ダッシュボード内でCTOに以下を依頼：
```
以下の5つのエージェントを起動してテストしてください：
1. Researcher（技術調査タスク）
2. Backend Coder（簡単な実装タスク）
3. Tester（テスト作成タスク）
4. Code Reviewer（コードレビュータスク）
5. Planner（タスク分解タスク）

各エージェントに簡単なタスクを割り当てて、正常に動作することを確認してください。
```

この方法なら：
- ✅ エージェント一覧に表示される
- ✅ ログがリアルタイムで表示される
- ✅ `StateManager`に自動登録される
